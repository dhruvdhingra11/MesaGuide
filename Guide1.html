Understanding the Miro API Integration

The Three API Tools

  1. CREATE BOARD: Makes a new Miro board
  2. CREATE SHAPE ITEM: Adds shapes (boxes, circles) to the board
  3. CREATE CONNECTOR: Draws lines between shapes

The AI agent decides when and how to call each tool based on the GTM plan.

Tool 1: CREATE BOARD

1Add HTTP Request Tool

  1. Click "+" to add a node
  2. Search for "HTTP Request Tool"
  3. Rename to: "CREATE BOARD"

2Configure Tool Description

This tells the AI when to use this tool:

Creates a new board on Miro for the mindmap.
Required parameter: board_name (string)

3Configure HTTP Request

Method: POST

URL:

https://api.miro.com/v2/boards

Headers:

Name Value
Authorization Bearer YOUR_MIRO_ACCESS_TOKEN
‚ö†Ô∏è REPLACE: YOUR_MIRO_ACCESS_TOKEN with your actual token from prerequisites

Body Type: JSON

JSON Body:

{
  "name": "{{ $fromAI('board_name') }}",
  "teamId": "YOUR_TEAM_ID",
  "policy": {
    "sharingPolicy": {
      "access": "view"
    }
  }
}
‚ö†Ô∏è REPLACE: YOUR_TEAM_ID with your Team ID from prerequisites
üîç Understanding this API Call
  • Endpoint: /v2/boards creates a new board
  • Authorization Header: Bearer token authenticates your request
  • $fromAI('board_name'): AI provides the board name dynamically
  • teamId: Specifies which Miro team owns this board
  • sharingPolicy: Sets access permissions (view/edit)

Response Example:

{
  "id": "uXjVGOly-mw=",
  "name": "Company GTM Plan",
  "teamId": "3458764655803963225"
}

The AI stores the id to use in subsequent calls.

Tool 2: CREATE SHAPE ITEM

1Add HTTP Request Tool

  1. Add another "HTTP Request Tool"
  2. Rename to: "CREATE SHAPE ITEM"

2Configure Tool Description

Creates a shape on a Miro board.

Required parameters (EXACT format):
- board_id: string (e.g. "uXjVGOly-mw=")
- shape_content: string (e.g. "Target Market")
- shape_type: string (must be: rectangle, round_rectangle, circle, rhombus)
- fill_color: string (e.g. "#8fd14f")
- position_x: number (e.g. 100)
- position_y: number (e.g. 0)
- width: number (e.g. 200)
- height: number (e.g. 100)

CRITICAL: position_x, position_y, width, height must be numbers without quotes.

3Configure HTTP Request

Method: POST

URL:

https://api.miro.com/v2/boards/{{ $fromAI('board_id') }}/shapes/

Headers:

Name Value
Authorization Bearer YOUR_MIRO_ACCESS_TOKEN

JSON Body:

{
  "data": {
    "content": "{{ $fromAI('shape_content') }}",
    "shape": "{{ $fromAI('shape_type') }}"
  },
  "style": {
    "fillColor": "{{ $fromAI('fill_color') }}",
    "borderColor": "#1a1a1a",
    "borderOpacity": "1.0",
    "borderWidth": "2",
    "color": "#1a1a1a",
    "fontFamily": "arial",
    "fontSize": "14",
    "textAlign": "center"
  },
  "position": {
    "x": {{ $fromAI('position_x') }},
    "y": {{ $fromAI('position_y') }}
  },
  "geometry": {
    "width": {{ $fromAI('width') }},
    "height": {{ $fromAI('height') }}
  }
}

‚ö†Ô∏è CRITICAL: Data Types Matter!

Notice the difference:

  • Strings (with quotes): "{{ $fromAI('shape_content') }}"
  • Numbers (NO quotes): {{ $fromAI('position_x') }}

If you add quotes around numbers, the API will fail!

üîç Understanding this API Call
  • Dynamic URL: Board ID is inserted from AI's decision
  • data.content: The text displayed in the shape
  • data.shape: Type (rectangle, circle, rhombus, round_rectangle)
  • style: Visual appearance (colors, borders, fonts)
  • position: X,Y coordinates on the canvas (0,0 is center)
  • geometry: Size of the shape in pixels

Example AI Call:

board_id = "uXjVGOly-mw="
shape_content = "Target Market"
shape_type = "round_rectangle"
fill_color = "#0CA789"
position_x = 400
position_y = 0
width = 220
height = 110

Response:

{
  "id": "3458764655804012345",
  "type": "shape",
  "data": {
    "content": "Target Market",

Understanding Tool Orchestration

How the AI Uses Multiple Tools

  1. Receives GTM Plan: AI reads the strategy
  2. Plans Structure: Decides what to visualize
  3. Creates Board: Calls CREATE BOARD tool
  4. Stores Board ID: Remembers ID for next calls
  5. Creates Central Shape: Calls CREATE SHAPE with position (0,0)
  6. Stores Shape ID: Remembers for connectors
  7. Creates Primary Branches: Multiple CREATE SHAPE calls
  8. Creates Secondary Nodes: More CREATE SHAPE calls
  9. Draws Connections: Multiple CREATE CONNECTOR calls
  10. Returns Result: Provides board URL and summary
